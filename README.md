### TechSupport Pro

**TechSupport Pro** — это локальная панель для техподдержки Windows: один красивый дашборд в браузере, который запускает подготовленные PowerShell‑скрипты (очистка, диагностика, безопасность, сеть и т.д.).

Приложение работает **целиком локально**, без интернета и установки сторонних программ.

---

## Возможности

- **Очистка системы**
  - Очистка `%TEMP%`, `C:\Windows\Temp`, `Prefetch`
  - Очистка корзины и кэша популярных браузеров
  - Очистка кеша обновлений Windows

- **Диагностика**
  - Информация о системе (CPU, RAM, диск, ОС)
  - Отчёт о здоровье дисков (S.M.A.R.T.)
  - Отчёт о батарее (`battery-report`)
  - Просмотр последних ошибок в журнале событий
  - Сводная диагностика системы

- **Сеть**
  - Сброс DNS‑кэша
  - Обновление IP‑адреса (renew)
  - Сброс Winsock / сетевого стека
  - Пинг до Google и простой тест скорости
  - Просмотр открытых портов и активных соединений

- **Безопасность**
  - Статус брандмауэра и Windows Defender
  - Запуск Windows Security (Defender)
  - Быстрая антивирусная проверка (Quick Scan)
  - Аудит безопасности (Firewall, Defender, UAC, Windows Update)

- **Инструменты**
  - Быстрый запуск: Диспетчер задач, Диспетчер устройств, Панель управления, MSConfig, RegEdit, PowerShell
  - Список процессов, служб, автозагрузки, установленных программ и т.п.

- **История, профили и бэкапы**
  - История выполненных команд
  - Профили («Быстрая очистка», «Полная очистка», «Исправление сети»)
  - «Псевдо‑бэкапы» состояния (системная и сетевая информация) с локальным хранением в браузере

## Быстрый запуск (для самых неопытных)

### 1. Скачать/скопировать папку проекта

Структура папки должна выглядеть примерно так:

```text
techsupport/
  start.bat
  server.ps1
  index.html
  script.js
  styles.css
  header.mp4 (опционально)
  modules/
    commands-system.ps1
    commands-network.ps1
    commands-management.ps1
    commands-info.ps1
    commands-security.ps1
```

Главное — чтобы **`start.bat` и `server.ps1` лежали рядом**.

---

### 2. Запуск обычный (без админа)

1. Откройте проводник и перейдите в папку `techsupport`.
2. Дважды щёлкните по файлу **`start.bat`**.
3. Откроется чёрное окно консоли, появится надпись вроде:

   ```text
   TechSupport Pro - Техническая поддержка
   Запуск локального веб-сервера...
   Приложение будет доступно по адресу: http://localhost:8080
   Для остановки сервера нажмите Ctrl+C
   ```

4. **Не закрывайте** это окно — это и есть запущенный сервер.
5. Откройте браузер (Chrome / Edge / другой) и введите в адресной строке:

   ```text
   http://localhost:8080
   ```

6. Откроется панель TechSupport Pro.

---

### 3. Запуск с правами администратора

Некоторые функции (например, очистка `C:\Windows\Temp`, `Prefetch`, MSConfig, быстрые сканы Defender) требуют админ‑прав.  

Чтобы их включить:

1. В проводнике **щёлкните правой кнопкой по `start.bat`**.
2. Выберите **«Запуск от имени администратора»**.
3. В появившемся окне контроля учётных записей нажмите **«Да»**.
4. Дальше всё как обычно:
   - В консоли появится информация о запуске
   - Откройте `http://localhost:8080` в браузере
   - В интерфейсе статус прав должен отображаться как «Права администратора»

## Типичные проблемы и решения

### 1. В браузере ничего не открывается

- Убедитесь, что окно консоли с сервером **открыто** и там нет красной ошибки.
- Проверьте, что адрес введён **точно**: `http://localhost:8080`
- Если порт 8080 занят:
  - Скрипт сам попробует найти свободный порт в диапазоне 8080–8090
  - Но в заголовке консоли URL пока жёстко указан как `http://localhost:8080` — если что, посмотрите в вывод сервера, на каком порту он всё-таки стартовал.

### 2. Сообщение «PowerShell не найден»

- Такое может быть на сильно кастомизированных системах.
- Откройте **Пуск → PowerShell** и убедитесь, что он запускается.
- Если путь до `powershell.exe` убран из `PATH`, верните его или запускайте сервер через PowerShell вручную:

```powershell
cd "путь\к\папке\techsupport"
powershell -ExecutionPolicy Bypass -File ".\server.ps1"
```

### 3. Ошибка «Admin required» в результатах команд

- Это **нормально**: команда требует прав администратора.
- Запустите `start.bat` **от имени администратора** и попробуйте ещё раз.

## 📝 Пояснения для каждой функции

### Очистка TEMP
Удаляет временные файлы из папки `%TEMP%`. Эти файлы создаются программами для временного хранения данных и обычно безопасны для удаления. Освобождает место на диске.

### Очистка Windows Temp
Удаляет системные временные файлы из `C:\Windows\Temp`. Требует прав администратора. Содержит временные файлы системных процессов и обновлений.

### Очистка Prefetch
Удаляет файлы предзагрузки из `C:\Windows\Prefetch`. Windows использует их для ускорения запуска программ. После удаления первый запуск программ может быть медленнее.

### Очистка корзины
Безвозвратно удаляет файлы из корзины. После этой операции восстановить удалённые файлы будет невозможно!

### Сброс DNS
Очищает локальный кэш DNS. Полезно когда сайт не открывается, но интернет работает. Заставляет систему заново запросить IP-адреса сайтов.

### Обновление IP
Выполняет `ipconfig /release` и `/renew`. Получает новый IP-адрес от DHCP-сервера. Помогает при IP-конфликтах.

### Сброс Winsock
Восстанавливает каталог Winsock, который отвечает за сетевые соединения. Решает проблемы с сетью после установки некоторых программ.

### SFC Scan
Проверяет целостность системных файлов Windows и восстанавливает повреждённые из кэша. Занимает 5-15 минут.

### CHKDSK
Проверяет диск на ошибки файловой системы. Может запланировать проверку при перезагрузке.
